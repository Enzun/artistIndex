■ アーティスト追加の流れ

    【JSONから一括登録（通常ルート）】
    1. data/artists.json にアーティスト名とチャンネルIDを追加
           フォーマット:
           [
             { "name": "YOASOBI", "channel_id": "UCvP_..." },
             ...
           ]
    2. 一括登録スクリプトを実行（collecting モードで登録される）
           npx tsx --env-file=.env scripts/bulk_add_artists.ts data/artists.json
    3. 実行後、data/bulk_add_report_YYYYMMDD-HHMMSS.json にレポートが保存される
           - added:   追加成功
           - skipped: 無効ID / 登録者不足 / 登録者非公開（重複は記録しない）
    4. 翌日から自動でデータ収集が始まる

    【スキップ基準】
    - 重複（すでにDB登録済みのチャンネルID）
    - 無効ID（YouTube APIに存在しないID）
    - 登録者非公開
    - 登録者20万人未満

    【1人だけ追加する場合】
    - collecting モード（データ収集のみ）
          npx tsx --env-file=.env scripts/add_artist.ts <channel_id> "アーティスト名"
    - active モード（即公開）
          npx tsx --env-file=.env scripts/add_artist.ts <channel_id> "アーティスト名" --active

    【過去の月次データを入れたい場合（SocialBlade等）】
    - data/<name>_monthly.json を用意（フォーマットは data/mga_monthly.json 参照）
    - dry-run で確認してから本実行
          npx tsx --env-file=.env scripts/import_social_blade.ts <artist_id> --file data/<name>_monthly.json --dry-run
          npx tsx --env-file=.env scripts/import_social_blade.ts <artist_id> --file data/<name>_monthly.json

■ アーティスト公開の流れ

    1. collecting 登録後、1〜2週間データを溜める
    2. Admin ページ（/admin）でアーティストを選んでグラフを確認
    3. 指数の動きが問題なければ「公開する」ボタンを押す → active に切り替わる
    4. /preview と / のアーティスト一覧に表示される

    ※ Admin パスワードは ADMIN_PASSWORD 環境変数で設定
